# 记忆数字练习功能设计

## 概述

为 Clock 7simul Flip 学习助手新增"记忆练习"模式。用户看到打乱文本后，通过长按空格启动计时，输入记忆码并确认提交，系统校验正确性并记录成绩，提供成绩列表与统计分析。

## 模式切换

- 页面顶部标题下方增加 **tab bar**，两个选项：学习模式（默认）/ 记忆练习
- 切换通过 CSS `display: none` 控制两个容器可见性
- 当前模式保存到 `localStorage`（`clock.ui.mode`）
- 切换到学习模式时移除计时器相关键盘监听

## 练习模式界面布局（从上到下）

1. **打乱文本** — 等宽字体，居中，醒目
2. **计时器显示** — 大字体居中，如 `0.000`
3. **状态提示** — 如"长按空格 2 秒准备..."
4. **输入区域** — 计时开始后出现，自动聚焦
5. **成绩统计面板** — best / mo3 / ao5 / ao12 / mean
6. **成绩列表** — 下方，倒序排列

## 计时器状态机

```
IDLE → HOLDING → READY → RUNNING → INPUT → FINISHED → IDLE
```

| 状态 | 触发条件 | 界面表现 |
|------|---------|---------|
| IDLE | 初始 / 上一轮结束 | 显示打乱，计时器归零，提示"长按空格准备" |
| HOLDING | 空格按下 | 计时器显示变红/黄，计算按住时长 |
| READY | 按住超过 2 秒 | 计时器显示变绿，提示"松开开始" |
| RUNNING | 松开空格 | 计时器开始计时，输入框出现并自动聚焦 |
| INPUT | 用户正在输入 | 计时持续，用户打字中 |
| FINISHED | 按下确认键 | 停止计时，校验，显示结果，记录成绩 |

特殊处理：

- HOLDING 不足 2 秒松开 → 回到 IDLE（防误触）
- RUNNING/INPUT 确认键在输入框内触发，不与空格启动冲突
- 确认键配置为 Space 时，输入框内空格变为提交键（记忆码不含空格）

从 FINISHED 自动生成下一条打乱，回到 IDLE。

## 记忆码校验

宽松匹配：忽略大小写和空格，提取所有非空白字符逐个比较。

示例：正确答案 `3 A 2 0 1 B`，以下输入均有效：
- `3A201B`
- `3 a 2 0 1 b`
- `3a201b`
- `3 A 2 0 1 B`

## 成绩数据

### 单条记录结构

```js
{
  id: 1713000000000,        // 时间戳 ID
  scramble: "UR3- DR2+ ...", // 打乱文本
  expected: "3 A 2 0 1 B",  // 正确记忆码
  input: "3a201b",           // 用户输入原文
  time: 4523,                // 毫秒
  valid: true,               // 记忆码是否正确
  date: "2026-04-12T..."     // ISO 时间
}
```

### 统计指标

- **best**：有效成绩中最佳单次
- **mo3**：最近 3 次有效成绩的平均
- **ao5**：最近 5 次有效成绩，去头尾取中间 3 个平均
- **ao12**：最近 12 次有效成绩，去头尾取中间 10 个平均
- **mean**：全部有效成绩平均
- 无效成绩标记为 **DNF**，不计入统计

### 成绩列表

- 倒序排列（最新在前）
- 每条显示：序号、时间（或 DNF）、日期
- 打乱默认隐藏，点击展开查看：打乱文本 + 正确记忆码 + 用户输入
- "清空全部成绩"按钮，带二次确认

### 持久化

`localStorage` key: `clock.memoTrainer.results`

## 快捷键与配置

| 操作 | 默认快捷键 | 说明 |
|------|-----------|------|
| 准备/开始计时 | 空格长按 2 秒 + 松开 | 全局监听，非输入框状态 |
| 停止计时并提交 | Enter | 输入框内触发 |

可配置确认键：Enter（默认）或 Space，通过下拉选择切换，保存到 `localStorage`（`clock.memoTrainer.confirmKey`）。

## 文件结构与模块职责

| 文件 | 变更 |
|------|------|
| `index.html` | 增加 tab bar、练习模式 DOM 容器、相关样式 |
| `app.js` | 增加模式切换逻辑，练习模式 UI 渲染与事件绑定 |
| `memo-trainer.js`（新建） | 计时器状态机、成绩管理、记忆码校验 |
| `clock-scramble.js` | 不变 |

### memo-trainer.js 导出

- `createTimerStateMachine()` — 管理状态流转
- `validateMemo(input, expected)` — 宽松匹配校验
- `loadResults() / saveResult(record) / clearResults()` — localStorage 读写
- `computeStats(results)` — 计算 best、mo3、ao5、ao12、mean

### app.js 职责

- 模式切换 DOM 控制
- 练习模式 UI 渲染
- 键盘事件注册/注销
- 调用 memo-trainer.js API

## 键盘事件处理策略

- IDLE/HOLDING/READY：`document` 上监听 `keydown`/`keyup`
- RUNNING/INPUT：输入框获得焦点，确认键在输入框事件中拦截
- 切换到学习模式时移除所有计时器相关键盘监听
